# Python Style Rules

When working with Python code in this project:

1. Use type hints for all function parameters and return values
2. Follow PEP 8 naming conventions
3. Use f-strings for string formatting
4. Prefer list comprehensions over map/filter when readable
5. Always include docstrings for public functions
